# .claudefiles Directory

This directory contains AI-generated content files created during development, testing, and example generation. These files are separate from the project source code.

## Directory Structure

```
.claudefiles/
├── examples/          # Example songs, lyrics, and use cases
├── templates/         # Reusable song templates and prompt templates
├── outputs/           # Tool execution outputs and analysis results
├── plans/             # Project plans, roadmaps, feature designs
├── ideas/             # Brainstorming, concepts, and suggestions
└── scratch/           # Temporary and experimental files
```

## Purpose

**Keep AI-generated content organized and separate from source code:**
- ✅ Easy to find all examples in one place
- ✅ Simple to clean up temporary files
- ✅ Share examples without cluttering the repository
- ✅ Track project evolution through plans and ideas
- ✅ Gitignore experimental work if needed

## Usage Guidelines

### For AI Assistants

**When to use `.claudefiles/`:**
- ✅ Creating example songs or lyrics
- ✅ Generating test data
- ✅ Saving tool execution results
- ✅ Creating templates or guides
- ✅ Writing project plans or roadmaps
- ✅ Brainstorming ideas or concepts
- ✅ Designing new features or architectures
- ✅ Experimental or temporary files

**When NOT to use `.claudefiles/`:**
- ❌ Source code files (use `src/`)
- ❌ Configuration files (use project root)
- ❌ Documentation (use `.github/` or root)
- ❌ Tests (use `tests/` or `__tests__/`)

### For Developers

**Organizing outputs:**
```bash
# Group by tool
.claudefiles/outputs/extract_song_dna/
.claudefiles/outputs/constraint_generator/

# Or by project/session
.claudefiles/outputs/concept-album-project/
.claudefiles/outputs/2025-10-sessions/

# Plans and ideas by topic
.claudefiles/plans/phase-2/
.claudefiles/ideas/visualization/
```

**Cleaning up:**
```bash
# Remove all scratch files
rm -rf .claudefiles/scratch/*

# Archive old outputs
mv .claudefiles/outputs .claudefiles/archive-2025-10
mkdir .claudefiles/outputs

# Archive completed plans
mv .claudefiles/plans/completed .claudefiles/plans/archive/
```

## Gitignore Recommendations

Add to `.gitignore`:
```gitignore
# Temporary AI-generated files
.claudefiles/scratch/

# Personal outputs (optional)
.claudefiles/outputs/

# Keep examples and templates in repo
!.claudefiles/examples/
!.claudefiles/templates/
```

## Maintenance

**Regular Cleanup Schedule:**
- Weekly: Clear `scratch/` directory
- Monthly: Archive old `outputs/`
- Quarterly: Review and organize `examples/`

**Quality Standards:**
- All files should have descriptive names
- Include metadata (date, tool, purpose)
- Use consistent formatting (Markdown preferred)
- Add README files for complex directory structures

## Notes

This directory structure helps maintain a clean separation between:
- **Source Code**: The MCP server implementation
- **Documentation**: How to use the server
- **Generated Content**: Examples, outputs, and experiments

By keeping generated content here, the project root remains clean and focused on the actual codebase.

---

**Remember**: This is where creative output lives. The `src/` directory is for code that creates; `.claudefiles/` is for what gets created.
